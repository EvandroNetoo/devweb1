CSS Units Comparison

A visual guide to the differences between CSS measurement units

Absolute Units

Absolute units are fixed and do not change based on other elements or the viewport. They provide exact measurements.

px (pixels)

200px width

Represents a single dot on the screen.

cm (centimeters)

5cm width

Represents a physical centimeter.

mm (millimeters)

50mm width

Represents a physical millimeter.

in (inches)

2in width

Represents a physical inch (2.54cm).

pt (points)

144pt width

Represents 1/72 of an inch.

pc (picas)

12pc width

Represents 1/6 of an inch.

Relative Units

Relative units provide flexibility and responsiveness. They are based on other measurements, such as the size of the parent element, the root element, or the viewport.

em

Parent: font-size: 16px
2em width = 32px

Relative to the font-size of the nearest parent element.

rem

10rem width

Relative to the font-size of the root element (html).

% (percentage)

Parent: width: 300px
50% width = 150px

Relative to the size of the parent element.

Viewport Units

50vw width

vw: 1% of viewport width

10vh height

vh: 1% of viewport height

30vmin width

vmin: 1% of the smaller viewport dimension

30vmax width

vmax: 1% of the larger viewport dimension

Text-Related Units

10ch width

ch: Relative to width of the "0" character

10ex width

ex: Relative to x-height of the current font

5cap width

cap: Relative to height of capital letters

5ic width

ic: Relative to width of "æ°´" character

Line Height Units

2lh width

lh: Relative to line-height of element

2rlh width

rlh: Relative to line-height of root element
